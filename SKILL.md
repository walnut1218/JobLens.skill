# BOSS直聘 职位搜索与求职分析 Skill

## 环境要求
- Python 3.10+
- 项目依赖通过 `uv` 管理（skill 根目录内的 `pyproject.toml` / `uv.lock`）
- 需要 token 凭证文件，用 `login` 子命令获取

## 数据结构

爬虫输出 CSV 的列（UTF-8 BOM 编码，Excel 可直接打开）：

| 列名 | 说明 |
|------|------|
| 职位名称 | 岗位名称 |
| 公司 | 招聘公司 |
| 薪资 | 薪资描述 |
| 城市 | 工作城市 |
| 经验 | 经验要求 |
| 学历 | 学历要求 |
| 融资阶段 | 公司融资阶段 |
| 公司规模 | 公司规模 |
| 行业 | 所属行业 |
| 职位描述 | 完整职位描述原文 |
| 详情链接 | BOSS直聘职位详情页 URL |

## 交互式对话流程

### 第 0 步：安装完成 → 请求登录

Skill 被安装后（或首次触发时），Agent 必须主动询问用户：

> **安装完成，是否需要安排登录 BOSS直聘？**（登录后才能搜索职位）

- 用户回复"开始登录"或其他确认 → 进入第 1 步
- 用户回复"不需要"或"跳过" → 跳到第 2 步（但需告知：不登录将无法采集职位数据）

### 第 1 步：执行登录

Agent 运行 login 命令：

```bash
uv run zhipin-search login -o ./token.json
```

**注意事项：**
- 命令会打开登录页面，用户需扫码或手机登录
- 登录成功后会自动写入 `token.json`，Agent 确认文件存在后进入下一步
- 如果登录失败，提示用户重试或选择手动登录方式

### 第 2 步：询问简历上传

登录完成后，Agent 询问用户：

> **登录成功 ✅ 是否上传本人简历及作品集进行分析？**
> 
> 上传后，系统会自动分析每个岗位与你的简历的**匹配度（0-100分）**，并按分数降序输出报告，匹配度高的排在最前面。
> 
> 不上传也不影响基础分析和报告生成，只是跳过匹配度评分和排序环节。

**如果用户上传：**
- 让用户提供简历/作品集文本内容（可直接粘贴或通过文件上传）
- 将内容写入 `user_resume.txt`（或 `user_resume.docx`）供 `scripts/_resume_matcher.py` 自动评分
- 告知用户：报告会按匹配度排序，顶部显示 Top 匹配职位一览

**如果用户选择不上传：**
- 告知：好的，不做评估
- 内部记录 `has_resume = false`

### 第 3 步：询问搜索参数

Agent 询问用户：

> **现在请告诉我搜索需求：**
> 1. 搜索哪个城市？（如 杭州、北京）
> 2. 搜索什么岗位？（如 用户研究、前端开发）
> 3. 搜多少页？（建议 3~5 页，每页约 15 条）
> 4. 是否要加更多筛选？（可选）
>    - 薪资范围：3K以下 / 3-5K / 5-10K / 10-20K / 20-50K / 50K以上
>    - 经验要求：应届 / 1年以内 / 1-3年 / 3-5年 / 5-10年 / 10年以上
>    - 学历要求：大专 / 本科 / 硕士 / 博士
>    - 公司规模：0-20人 / 20-99人 / 100-499人 / 500-999人 / 1000-9999人 / 10000人以上
>    - 行业：互联网 / 游戏 / 人工智能 / 金融 / 教育培训 / 医疗健康 / 汽车 等
>    - 融资阶段：未融资 / 天使轮 / A轮 / B轮 / C轮 / D轮及以上 / 已上市 / 不需要融资
>    - 职位类型：（全职/兼职/实习——注意：BOSS 直聘兼职和实习共用同一筛选码，无法精确区分，建议直接用关键词搜索）
>    - （直接说不需要即可跳过）

等待用户回复参数后进入下一步。

**注意：** 如果用户不提供筛选条件，则只按关键词 + 城市搜索，不加额外过滤。

### 第 4 步：运行爬虫（由 Agent 自动执行）

爬虫脚本位于 `scripts/scrape_all.py`。

```bash
uv run python scripts/scrape_all.py --keyword <关键词> --city <城市> --pages <页数> \
  [--salary <薪资>] [--experience <经验>] [--education <学历>] \
  [--scale <公司规模>] [--industry <行业>] [--stage <融资阶段>]
```

全部可选筛选项（与第 3 步对应）：
- `--salary`：3K以下 / 3-5K / 5-10K / 10-20K / 20-50K / 50K以上
- `--experience`：应届 / 1年以内 / 1-3年 / 3-5年 / 5-10年 / 10年以上
- `--education`：大专 / 本科 / 硕士 / 博士
- `--scale`：0-20人 / 20-99人 / 100-499人 / 500-999人 / 1000-9999人 / 10000人以上
- `--industry`：互联网 / 游戏 / 人工智能 / 金融 / 教育培训 / 医疗健康 / 汽车 等
- `--stage`：未融资 / 天使轮 / A轮 / B轮 / C轮 / D轮及以上 / 已上市 / 不需要融资

脚本将输出 CSV 文件到 `./jobs_{城市}_{关键词}_详情.csv`。

**注意：** 爬虫使用真实 Chrome 浏览器（防风控），会弹出浏览器窗口。

### 第 5 步：读取爬取数据

爬虫完成后，Agent 读取生成的 CSV 文件。CSV 为 UTF-8 BOM 编码，读取时指定 `encoding='utf-8-sig'`。

### 第 5a 步：去重过滤（反复搜索不重复分析）

Agent 必须运行去重脚本，过滤掉历史已见过的职位：

```bash
uv run python scripts/_dedup.py "jobs_{城市}_{关键词}_详情.csv"
```

**逻辑：**
- 从 CSV 每个职位的 `详情链接` URL 提取唯一 `encryptJobId`
- 与 `seen_jobs.txt`（已见 ID 记录本）对比
- 已见的 → 跳过；新出现的 → 保留，追加到 `seen_jobs.txt` 并写入 `jobs_data.json`

**去重后判断：**
- `jobs_data.json` 有新职位 → 进入第 6 步
- 无新职位 → 告知用户"今天还没有出现新职位"，流程结束

> `seen_jobs.txt` 是持久文件，每天累积。跨城市、跨关键词可分不同文件独立管理。

### ⚠️ 硬性规则（不得违反）

Agent 必须遵守以下硬性规则：

#### 格式与完整性

1. **格式锁定**：必须严格遵循 `references/analysis_prompt.md` 的模板结构，不得添加/遗漏/合并。
2. **先报告后建议**：先全量输出所有职位的结构化分析，再给总结建议，严禁跳过。
3. **JD原文必带**：每条分析必须展示**职位描述原文**（从 {description} 字段获取），不得省略。
4. **岗位链接必带**：每条分析必须展示**岗位链接**（从 {url} 字段获取），方便用户直接打开查看。
5. **多条先总结**：当职位数 > 10 条时，必须在逐条分析前先生成**整体总结**（按 analysis_prompt.md 第一部分模板），让用户先看全景再深入。
6. **JD缺失**：标注"信息不足"，不得编造。
7. **输出格式**：默认输出 Word（.docx），逐条写入，含完整模板内容。
8. **报告路径告知**：报告生成后，必须明确告知用户文件的**完整绝对路径**，方便直接打开。格式示例：`📄 报告已生成：C:\Users\xxx\...\分析报告_杭州_用户研究实习生_20260514.docx`

#### 深度分析要求（核心——这是求职顾问，不是数据提取器）

9. **必须读出 JD 背后的信息**：每条分析必须包含"深度解读"——从 JD 措辞推断团队状态、岗位定位、JD 里没写的真实问题。不得只是翻译 JD。
10. **能力谱系三档**：每个岗位必须区分"门槛级/竞争级/溢价级"三个能力层次，不是简单罗列技能名词。
11. **技能必须带实现路径**：每项核心技能必须写明：到什么程度够用、最快上手路径、加分操作、避坑点。不得只写"需要 XX"。
12. **作品集必须具体到数据来源**：必须写明项目目标、具体内容、数据来源、技术栈、预期产出、完成时间、对应 JD 的哪几条。不得写"做一个 XX 项目"就完事。
13. **面试题必须是真实场景**：针对这家公司/这个方向设计场景题，必须写明面试官的真实意图、回答的核心逻辑、加分操作。不得出模板题（如"请自我介绍"）。
14. **必须给判断，不要中立**：如果这个岗位不值得投，直接说。如果某条建议有优先级，明确排出来。不允许每条都"建议可以考虑"。
15. **简历匹配必须有行动方案**：匹配分析不能只写"匹配/不匹配"，必须给"一周突击方案"——具体做什么来弥补硬伤。

### 第 6 步：加载分析提示词

读取 `references/analysis_prompt.md` 获取分析提示词模板。

**⚠️ 这是整个 skill 的核心。** analysis_prompt.md 定义了分析的深度和视角——你不是在翻译 JD，你是在解读 JD。你必须像一个比自己资深 5 年的同行在给用户提建议：务实、直接、有判断力。

**关键判断：**
- 如果 `user_resume.txt` 存在且非空 → `generate_report.py` 自动计算匹配度并排序（详情见下方"简历匹配度评分说明"）
- 如果用户未上传简历 → 按照标准提示词输出，不追加匹配度评估部分

### 第 7 步：读取 jobs_data.json 并逐条分析

读取 **`jobs_data.json`**（仅包含本次新增的职位）进行分析。

**如果职位数 > 10 条，先输出整体总结：**

按 `analysis_prompt.md` 第一部分的模板，扫描所有职位数据，生成：
- 📊 数据概览（行业分布、薪资区间、经验要求）
- 🎯 高频核心能力 TOP 5
- 📁 作品集方向建议
- 💡 求职策略建议

再逐条分析每个职位。

对每一条职位数据，依次应用 analysis_prompt.md 第二部分的提示词，输出结构化分析。

**⚠️ 分析深度要求：**
- **深度解读**：必须从 JD 措辞推断团队状态、岗位定位、隐藏问题——这些信息 JD 里没有直接写，需要你基于行业经验推导
- **能力谱系**：不是罗列技能名词，而是分"门槛级/竞争级/溢价级"三档，每档写清"到什么程度""怎么达到"
- **技能实现路径**：每项技能必须写"够用程度 → 最快上手路径 → 加分操作 → 避坑"，不允许只写"需要XX"
- **作品集方案**：必须具体到数据来源、技术栈、完成时间、对应 JD 哪几条——不是"做一个XX项目"
- **面试模拟**：出这家公司特有的场景题，不是模板题，必须写面试官意图+回答逻辑+加分操作

**注意替换占位符：** analysis_prompt.md 模板中的 `{url}`、`{description}`、`{total}` 是占位符，必须用实际数据替换（`{total}` 替换为本次职位总数）。

**每条职位必须独立分析，不要合并或交叉引用。** 按 JSON 中的顺序逐条输出。

输出为 **Word（.docx）** 格式文档，文件名：`分析报告_{城市}_{关键词}_{日期}.docx`。

使用 Python 脚本（python-docx）生成，确保：
- 中文字体设置为微软雅黑或宋体
- 标题层级清晰（H1 → 报告标题，H2 → 整体总结，H3 → 各职位）
- 整体总结放在最前面，其后逐条展示各职位分析
- 每个职位完整包含分析模板的全部内容（含 JD原文 和 岗位链接）
- 有序号递增（职位 1、职位 2、...）
- 保存到 skill 根目录

**报告生成后，必须主动告知用户文件路径**，例如：

> 📄 报告已生成 → `C:\Users\xxx\.qclaw\skills\boss-job-analyzer\分析报告_杭州_用户研究实习生_20260514.docx`

### 第 8 步（无简历兜底）：询问是否需要简历上传

如果用户在第 2 步选择了**不上传**，则在完成所有职位分析输出后，Agent 主动询问：

> **所有职位分析已输出完毕 📊**
> 
> 如需进一步评估这些岗位与您的匹配度，可以上传您的简历/作品集，我可以针对每个职位重新评估：
> - ✅ 匹配的点
> - ❌ 不匹配的点
> - 📝 简历/作品集修改建议
> 
> 需要上传分析匹配度吗？

- 用户同意上传 → 执行第 2 步的上传逻辑 → 重新输出带匹配度评估的分析
- 用户拒绝 → 结束流程

## 简历匹配度评分说明

`scripts/_resume_matcher.py` 读取 `user_resume.txt`，对每个职位按以下维度自动打分（0~100）：

| 维度 | 分值 | 说明 |
|------|------|------|
| 岗位名称匹配 | 20分 | 职位名是否含"用户研究"/"UX"/"用研"等关键词 |
| 学历匹配 | 10分 | 硕士 vs 岗位要求的学历匹配度 |
| 经验匹配 | 10分 | 应届/实习 vs 岗位经验要求 |
| 技能匹配 | 25分 | JD中出现 Python/NLP/SPSS/HLM/爬虫 等技能 |
| 研究领域匹配 | 15分 | 研究方向（社交媒体、用户行为等）与 JD 重合度 |
| 行业匹配 | 10分 | 互联网/游戏/社交 等与用户背景结合的行业 |
| 反相关降分 | -10分 | 岗位是开发/销售/运营等非用研方向 |

总分 0~100，≥60 为高匹配。如果 `user_resume.txt` 不存在，自动跳过评分，按原始顺序输出。

**简历文件位置**：skill 根目录的 `user_resume.txt`（示例格式见下方）。

```text
学历：本科
专业：语言学
学校：华中科技大学
技能：Python, NLP, 爬虫, 数据分析, HLM, SPSS, 情感分析
研究方向：社交媒体分析, 用户行为, 情绪分析
经验：0-1年
目标岗位：用户研究, 用户研究员, 数据分析师
```

### 关于风控说明

如果爬取到的数据量较少（少于预期页数），可能是触发了 BOSS直聘的反爬机制。属正常现象，已获取到的数据足够分析。

## 每日自动推送

Skill 内置一个定时任务，每天 **9:00（北京时间）** 自动检查新岗位。

### 工作方式

1. **定时触发**：每天 9:00 自动运行 `push_daily.py`
2. **Headless 爬取**：使用真实 Chrome 但无头模式（尽力尝试，可能受风控影响）
3. **自动去重**：比对 `seen_jobs.txt`，新职位写入 `jobs_data.json`
4. **结果输出**：
   - 有新职位 → 告知数量 + 摘要（公司/岗位/薪资）
   - 无新职位 → 告知「今日无新增岗位」
   - 爬取失败（风控） → 明日自动重试

### 手动触发

用户也可以主动触发每日检查。当用户说「检查今天的新岗位」或「每日推送」时，Agent 运行：

```bash
uv run python scripts/push_daily.py --city <城市> --keyword <关键词> --pages <页数> \
  [--salary <薪资>] [--experience <经验>] [--education <学历>] \
  [--scale <公司规模>] [--industry <行业>] [--stage <融资阶段>]
```

### 用户主动触发（推荐）

如果当前浏览器会话可用（headless=False），用户可让 Agent 手动执行完整爬取流程以获得最佳效果（绕过风控），流程自动走第 1~5a 步的去重逻辑，仅分析新职位。
